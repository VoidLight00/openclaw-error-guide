1
OpenClaw이 뭔가요?

OpenClaw은 내 컴퓨터에서 실행되는 개인 AI 비서입니다. 텔레그램, WhatsApp, Discord 같은 메신저를 통해 AI와 대화하면서 실제 작업(파일 관리, 웹 검색, 브라우저 제어, 일정 확인 등)을 수행할 수 있습니다.

ChatGPT / Claude와 뭐가 다른가요?
ChatGPT나 Claude는 웹 브라우저에서 대화만 가능하지만, OpenClaw은 내 컴퓨터에서 직접 실행되기 때문에 파일을 읽고 쓰고, 명령어를 실행하고, 웹을 검색하고, 브라우저를 제어하는 등 실제 작업이 가능합니다.

핵심 개념 3가지

게이트웨이

OpenClaw의 핵심 서버. 항상 백그라운드에서 실행되며 메시지를 받고, AI를 호출하고, 작업을 수행합니다.

에이전트

AI 비서 그 자체. 이름, 성격, 기억을 가지며 워크스페이스에서 일합니다.

세션

하나의 대화 맥락. 매일 오전 4시에 자동 리셋되거나 /new로 새 대화를 시작할 수 있습니다.

전체 구조 한눈에 보기

Architecture 
┌─────────────────────────────────────────┐
│  사용자 (텔레그램 / TUI / 대시보드)         │
└──────────────┬──────────────────────────┘
               │ 메시지
               ▼
┌─────────────────────────────────────────┐
│  게이트웨이 (Gateway)                      │
│  - 메시지 수신/발신                         │
│  - AI 모델 호출                            │
│  - 스킬/도구 실행                           │
│  - 크론잡/하트비트 관리                      │
└──────────────┬──────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────┐
│  워크스페이스 (~/.openclaw/workspace)       │
│  - SOUL.md, IDENTITY.md, USER.md        │
│  - MEMORY.md, memory/ 폴더               │
│  - AGENTS.md, TOOLS.md, HEARTBEAT.md    │
└─────────────────────────────────────────┘
2
워크스페이스 파일 이해하기

워크스페이스는 AI 비서의 "작업 공간"입니다. 기본 위치는 ~/.openclaw/workspace이며, 여기에 있는 마크다운 파일들이 AI의 성격, 기억, 행동 규칙을 결정합니다.

왜 중요한가요? AI는 매 세션마다 이 파일들을 읽고 "나는 누구인지, 사용자는 누구인지, 어떻게 행동해야 하는지"를 파악합니다. 파일을 수정하면 AI의 행동이 달라집니다!

SOUL.md — AI의 영혼

AI의 핵심 성격, 행동 원칙, 경계선을 정의합니다. "진심으로 도움이 되어라", "의견을 가져라" 같은 기본 가치가 담겨 있습니다.

IDENTITY.md — AI의 신분증

AI의 이름, 종류(AI? 로봇? 정령?), 분위기, 대표 이모지를 기록합니다. 처음 대화할 때 AI와 함께 정합니다.

USER.md — 사용자 프로필

사용자의 이름, 호칭, 타임존, 관심사 등을 기록합니다. AI가 사용자를 더 잘 이해하는 데 사용됩니다.

MEMORY.md — 장기 기억

AI가 직접 정리하는 장기 기억 파일입니다. 중요한 결정, 선호도, 배운 것들이 기록됩니다. 메인 세션에서만 로드됩니다.

AGENTS.md — 운영 매뉴얼

AI의 행동 규칙 전체를 담은 파일입니다. 기억 관리 방법, 안전 수칙, 그룹 채팅 에티켓 등이 정의되어 있습니다.

HEARTBEAT.md — 주기적 체크리스트

AI가 주기적으로(기본 30분마다) 확인해야 할 사항을 적는 파일입니다. 이메일 확인, 캘린더 체크 등을 적으면 AI가 알아서 확인합니다.

TOOLS.md — 환경 설정 메모

카메라 이름, SSH 호스트, TTS 음성 설정 등 내 환경에 특화된 정보를 기록하는 파일입니다.

memory/YYYY-MM-DD.md — 일일 기록

매일의 대화 기록과 메모가 저장됩니다. AI는 오늘과 어제의 기록을 세션 시작 시 자동으로 읽습니다.

팁: 이 파일들은 모두 일반 마크다운 파일이므로 텍스트 에디터로 직접 수정할 수 있습니다. 또는 AI에게 "SOUL.md를 더 캐주얼하게 바꿔줘"라고 말하면 AI가 직접 수정합니다!
3
기본 사용법

3-1. 텔레그램에서 대화하기

설치가 완료되었다면, 텔레그램에서 봇에게 메시지를 보내면 됩니다.

명령어설명
/new새 대화 시작 (이전 맥락 초기화)
/new opus새 대화 + 모델 변경
/stop현재 실행 중인 작업 중지
/model사용 가능한 모델 목록 보기
/model 3목록에서 3번 모델로 변경
/model status현재 모델 상세 정보
/status에이전트 상태 확인 (컨텍스트 사용량 등)
/compact오래된 대화 요약하여 컨텍스트 절약
/context list시스템 프롬프트에 뭐가 들어있는지 확인
/reasoning onAI의 사고 과정 보기

3-2. TUI (터미널 UI) 사용하기

터미널에서 직접 AI와 대화할 수 있는 인터페이스입니다.

Terminal 
openclaw tui

Ctrl+C로 종료합니다. 게이트웨이 URL과 토큰을 지정해서 원격 접속도 가능합니다.

3-3. 웹 대시보드 (Control UI)

브라우저에서 OpenClaw을 관리하고 대화할 수 있는 웹 인터페이스입니다.

Terminal 
openclaw dashboard

기본적으로 http://127.0.0.1:18789/ 에서 접속할 수 있습니다. 채널 설정 없이도 바로 대화할 수 있어서 가장 빠른 시작 방법입니다.

채널 설정 없이 바로 시작하려면?
openclaw dashboard 명령어로 웹 대시보드를 열면 텔레그램 설정 없이도 바로 AI와 대화할 수 있습니다!
4
AI 모델 설정

OpenClaw은 다양한 AI 모델을 지원합니다. 목적에 따라 모델을 바꿔가며 사용할 수 있습니다.

모델 확인 및 변경

Terminal 
# 현재 모델 및 인증 상태 확인
openclaw models status

# 기본 모델 변경
openclaw models set anthropic/claude-sonnet-4-5

# 채팅 중 모델 변경 (세션 내에서만)
# 텔레그램에서 /model 입력

지원하는 주요 AI 제공자

제공자대표 모델특징
AnthropicClaude Opus, Sonnet도구 사용 능력 뛰어남, 안전성 높음
OpenAIGPT-5.2, Codex범용, 코딩 강점
GoogleGemini무료 티어 있음
MiniMaxMiniMax-M2.1글쓰기 강점, 비용 효율적
OpenRouter여러 모델 중개다양한 모델 한 곳에서 사용

폴백(Fallback) 설정

메인 모델이 장애가 나면 자동으로 다른 모델로 전환되도록 설정할 수 있습니다.

Terminal 
# 폴백 모델 추가
openclaw models fallbacks add openai/gpt-5.2

# 폴백 목록 확인
openclaw models fallbacks list
모델 선택 팁:
- 일상 대화: Claude Sonnet (빠르고 저렴)
- 복잡한 분석: Claude Opus (느리지만 정확)
- 코딩: GPT-5.2 또는 Codex
- 비용 절약: MiniMax-M2.1 또는 OpenRouter 무료 모델
5
스킬 활용하기

스킬은 AI에게 새로운 능력을 가르치는 확장 기능입니다. 각 스킬은 SKILL.md 파일을 포함한 폴더로, AI가 특정 도구를 어떻게 사용해야 하는지 설명합니다.

비유하면: AI가 "비서"라면, 스킬은 비서가 배우는 "업무 매뉴얼"입니다. 이미지 생성 매뉴얼, 웹 검색 매뉴얼, TTS(음성 합성) 매뉴얼 등이 있습니다.

스킬 위치 (우선순위 순)

위치설명우선순위
<workspace>/skills내 워크스페이스 전용 스킬가장 높음
~/.openclaw/skills사용자 설치 스킬 (모든 에이전트 공유)중간
번들 스킬OpenClaw에 기본 내장된 스킬가장 낮음

스킬 관리 명령어

Terminal 
# 설치된 스킬 목록
openclaw skills list

# 스킬 설정 (API 키 등)
openclaw configure --section skills

ClawHub — 스킬 마켓

clawhub.com에서 커뮤니티가 만든 스킬을 검색하고 설치할 수 있습니다.

Terminal 
# 스킬 설치
clawhub install <skill-slug>

# 모든 스킬 업데이트
clawhub update --all

주요 내장 도구

도구설명설정 필요?
web_search웹 검색 (Brave Search API)API 키 필요
web_fetch웹 페이지 내용 가져오기기본 사용 가능
browser브라우저 자동 제어 (클릭, 입력 등)설정에서 활성화
exec터미널 명령어 실행기본 사용 가능
read/write/edit파일 읽기/쓰기/편집기본 사용 가능

웹 검색 설정하기

Terminal 
# 가장 쉬운 방법: 대화형 설정
openclaw configure --section web

# Brave Search API 키가 필요합니다
# https://brave.com/search/api/ 에서 무료로 발급

서브에이전트 (Sub-Agent)

메인 대화를 차단하지 않고 백그라운드에서 병렬 작업을 수행할 수 있습니다.

사용 예: "서브에이전트로 최신 Node.js 릴리즈 노트를 조사해줘" → AI가 백그라운드에서 조사하고 결과를 보고합니다. 그동안 메인 대화는 계속 가능!
6
자동화 기능

6-1. 하트비트 (Heartbeat)

기본 30분마다 AI가 자동으로 "깨어나서" 할 일이 있는지 확인합니다.

비유: 비서가 30분마다 "확인할 게 있나요?"하고 체크하는 것. 급한 일이 있으면 알려주고, 없으면 조용히 넘어갑니다.

HEARTBEAT.md에 체크리스트 추가하기:

HEARTBEAT.md 
# Heartbeat checklist
- 급한 이메일이 있는지 확인
- 2시간 이내 캘린더 일정 확인
- 진행 중인 작업 상태 체크

하트비트 설정 변경:

Terminal 
# 간격을 1시간으로 변경
openclaw config set agents.defaults.heartbeat.every "1h"

# 하트비트 끄기
openclaw config set agents.defaults.heartbeat.every "0m"

6-2. 크론잡 (Cron Job)

정확한 시간에 작업을 실행하는 예약 기능입니다.

사용 사례추천 방식이유
30분마다 이메일 확인하트비트다른 체크와 묶어서 처리
매일 아침 9시 리포트크론잡정확한 시간 필요
20분 뒤 알림크론잡 (--at)일회성 정확한 타이밍
주간 프로젝트 분석크론잡 (isolated)독립 세션, 다른 모델 사용 가능

크론잡 예시:

Terminal 
# 20분 뒤 알림 (일회성)
openclaw cron add \
  --name "미팅 알림" \
  --at "20m" \
  --session main \
  --system-event "알림: 10분 뒤 스탠드업 미팅!" \
  --wake now \
  --delete-after-run

# 매일 아침 7시 브리핑 (반복)
openclaw cron add \
  --name "아침 브리핑" \
  --cron "0 7 * * *" \
  --tz "Asia/Seoul" \
  --session isolated \
  --message "오늘 일정과 주요 이메일을 요약해줘" \
  --announce \
  --channel telegram \
  --to "내_텔레그램_ID"

# 크론잡 목록 확인
openclaw cron list

6-3. 훅 (Hooks)

특정 이벤트가 발생하면 자동으로 실행되는 스크립트입니다.

번들 훅설명
session-memory/new 입력 시 현재 대화를 메모리 파일로 저장
command-logger모든 명령어를 로그 파일에 기록
boot-md게이트웨이 시작 시 BOOT.md 실행
Terminal 
# 훅 목록 보기
openclaw hooks list

# 세션 메모리 훅 활성화 (추천!)
openclaw hooks enable session-memory

문제 해결
문제 발생 시 가장 먼저 실행할 명령어:
Terminal 
openclaw status            # 전체 상태 확인
openclaw gateway status    # 게이트웨이 상태
openclaw doctor            # 자동 진단 및 수리
openclaw logs --follow     # 실시간 로그 확인

텔레그램에서 봇이 응답하지 않아요

  1. 게이트웨이가 실행 중인지 확인: openclaw gateway status
  2. 페어링이 완료되었는지 확인: openclaw pairing list telegram
  3. 게이트웨이 재시작: openclaw gateway restart

게이트웨이가 중지되어 있어요

Terminal 
openclaw gateway start

Windows에서는 예약 작업으로 등록되어 있으므로 재부팅 후에도 자동 실행됩니다.

"Model is not allowed" 오류가 떠요

모델 허용 목록(agents.defaults.models)에 해당 모델이 없기 때문입니다.

Terminal 
# 채팅에서 사용 가능한 모델 확인
# /model list

# 또는 허용 목록 해제
openclaw config unset agents.defaults.models

대화 컨텍스트가 너무 커졌어요

텔레그램에서 /compact를 입력하면 오래된 대화를 요약해서 컨텍스트를 줄입니다. 또는 /new로 새 대화를 시작하세요.

하트비트 메시지가 30분마다 와요

정상 동작입니다! 줄이거나 끄려면:

Terminal 
# 간격 늘리기
openclaw config set agents.defaults.heartbeat.every "2h"

# 완전히 끄기
openclaw config set agents.defaults.heartbeat.every "0m"

명령어가 안 먹혀요 (PowerShell)

PowerShell을 닫고 새로 열어보세요. PATH 환경변수가 갱신되지 않았을 수 있습니다.

PowerShell 
# 실행 정책 문제일 경우:
Set-ExecutionPolicy RemoteSigned
