텔레그램 봇 연동 가이드

OpenClaw와 텔레그램 BotFather를 연결하는 완전 가이드

1
사전 준비
  • OpenClaw 최신 버전 설치 확인
  • Node.js 22 이상 설치 확인
  • 텔레그램 앱 설치 및 계정 활성화
Terminal 
npm install -g openclaw@latest
node --version   # v22.x 이상 확인
2
BotFather로 봇 생성
  • 텔레그램에서 @BotFather 검색 후 채팅 시작
  • /newbot 명령어 입력
  • 봇 이름 설정 (예: MyOpenClawBot)
  • 사용자 이름 설정 (예: myopenclaw_bot) -- 반드시 bot으로 끝나야 함
  • 발급된 토큰 (예: 123456:ABC-DEF...) 복사 후 안전하게 보관
보안 주의: 토큰이 유출되면 봇이 탈취될 수 있습니다. 절대 공개 저장소나 채팅에 토큰을 노출하지 마세요.
3
OpenClaw 설정 & 토큰 입력

방법 A: Claude Code로 설정 (추천)

  • Claude Code 터미널을 열고 아래처럼 입력하세요
Claude Code 
openclaw 텔레그램 봇 연동해줘. 
BotFather 토큰: 여기에_토큰_입력

Claude Code가 알아서 다음을 처리합니다:

  • ~/.openclaw/openclaw.json 파일을 읽고 텔레그램 채널 설정 추가
  • 게이트웨이 재시작 (openclaw gateway restart)
  • 연결 상태 확인
Claude Code가 OpenClaw 설정 파일 수정 권한을 요청하면 승인해주세요. 토큰만 준비되어 있으면 나머지는 자동입니다.

더 상세하게 지시할 수도 있습니다:

Claude Code (상세 버전) 
~/.openclaw/openclaw.json 파일에 텔레그램 채널을 추가해줘.
봇 토큰은 123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11 이야.
설정 후 openclaw gateway restart 실행하고 
openclaw status로 연결 상태 확인해줘.

방법 B: 대화형 설정 (터미널)

Terminal 
openclaw gateway stop
openclaw onboard
# Config handling → "Update values" 선택
# 채널 → "Telegram (Bot API)" 선택
# 토큰 입력

방법 C: OpenClaw 대시보드에서 설정

  • 브라우저에서 http://localhost:3000 접속 (OpenClaw 대시보드)
  • 좌측 메뉴에서 Settings (설정) 클릭
  • Channels (채널) 섹션으로 이동
  • Telegram 항목에서 Enabled 토글을 켜기
  • Bot Token 필드에 BotFather에서 받은 토큰 붙여넣기
  • Save (저장) 클릭 후 게이트웨이 자동 재시작
대시보드가 안 열리면 먼저 openclaw gateway start를 실행하세요. 대시보드 포트는 설정에 따라 다를 수 있습니다 (기본: 3000).

방법 D: 수동 설정 (JSON 직접 편집)

~/.openclaw/openclaw.json 파일에 직접 추가:

JSON 
{
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "여기에_토큰_입력"
    }
  }
}
4
페어링 코드 승인
  • OpenClaw 게이트웨이 시작
  • 텔레그램 봇에서 /start 입력 후 페어링 코드 확인
  • 터미널에서 페어링 코드 승인
  • 사용자 ID는 @userinfobot으로 확인 후 allowFrom에 추가
Terminal 
openclaw gateway start
# 텔레그램 봇에서 /start 입력 후...
openclaw pairing list telegram
openclaw pairing approve telegram [코드]
5
그룹 채팅 설정 (선택)
  • BotFather에서 /setprivacy 실행 후 Privacy Mode를 Disable로 변경
  • 봇을 그룹에 초대 (관리자 권한 추천)
  • OpenClaw 설정에서 그룹 모드 구성
JSON 
{
  "groups": {
    "*": {
      "requireMention": true
    }
  }
}
Privacy Mode 변경 후 봇을 그룹에서 제거했다가 다시 초대해야 설정이 적용됩니다.
6
문제 해결
증상 해결 방법
DM 무응답 토큰 재확인, openclaw health 또는 openclaw gateway --verbose 실행
그룹 무응답 멘션(@봇이름) 테스트, Privacy Mode 점검
충돌/초기화 rm -rf ~/.openclaw 후 재설치
페어링 안 됨 gateway 재시작, /start 재입력
추가 문제가 발생하면 채널 통합 오류 페이지를 참고하세요.
