OpenClaw Windows 설치 가이드

텔레그램 봇 생성부터 페어링까지, 한 번에 성공하세요.

시작하기 전에 -- 꼭 읽어주세요!
이 가이드는 실제 Windows 환경에서 설치하면서 겪은 시행착오를 바탕으로 작성되었습니다. 아래 3가지만 기억하면 한 번에 성공할 수 있습니다.

흔한 실수 3가지와 해결법

순서흔한 실수이렇게 하세요
1 npm i -g openclaw 실행 시 빌드 오류 발생 반드시 --ignore-scripts 옵션을 붙여서 설치
2 Gateway 서비스 설치 실패 처음부터 관리자 권한 PowerShell에서 모든 작업 진행
3 텔레그램에서 봇이 응답하지 않음 페어링(연결 인증) 과정을 빠뜨리지 않기
1
사전 준비
설치를 시작하기 전에 아래 항목을 모두 준비해두세요. 중간에 가입하거나 다운로드하면 흐름이 끊기고 시간이 오래 걸립니다.

1-1. 필요한 계정 (미리 가입)

계정용도가입 링크비용
Anthropic Claude Code 사용 + OpenClaw AI 모델 console.anthropic.com 유료
텔레그램 봇과 대화할 채팅 플랫폼 telegram.org 무료
Anthropic 비용 안내: Anthropic의 Claude Pro/Max 구독 ($20/월~) 또는 API 크레딧 충전이 필요합니다. Claude Code와 OpenClaw 모두 별도의 API Key 발급 없이, 터미널 명령어를 통해 인증을 처리합니다.
다른 AI 모델을 사용할 경우: OpenClaw은 Claude 외에도 GPT(OpenAI), Gemini(Google) 등을 지원합니다.
- OpenAI: platform.openai.com (유료)
- Google Gemini: aistudio.google.com (무료 티어 있음)

1-2. 발급이 필요한 토큰

항목용도발급 시점
텔레그램 봇 토큰 OpenClaw과 텔레그램 연결 설치 전 미리 발급 -- 2단계에서 안내
Anthropic 토큰 OpenClaw AI 모델 인증 설치 중 발급 -- 4단계에서 안내

1-3. 미리 설치할 소프트웨어

소프트웨어용도다운로드
텔레그램 앱 봇 생성 및 대화 desktop.telegram.org (PC) 또는 앱스토어 (모바일)
Git, Claude Code, Node.js는 아래 가이드에서 명령어로 설치합니다. 미리 다운로드할 필요 없습니다.

1-4. 시스템 요구사항

항목요구사항
운영체제Windows 10 이상 (Windows 11 권장)
관리자 권한PowerShell을 관리자 권한으로 실행할 수 있어야 함
인터넷 연결설치 및 AI 모델 호출에 필요
디스크 공간약 500MB 이상 여유 공간

1-5. 준비 체크리스트

  • Anthropic 계정 가입 완료 (Pro/Max 구독 또는 API 크레딧 충전)
  • 텔레그램 계정 가입 및 앱 설치
  • 텔레그램 봇 토큰 발급 및 저장 (@BotFather -> /newbot)
  • Windows 관리자 계정으로 로그인되어 있는지 확인

이 가이드의 진행 방식

이 가이드는 Claude Code(터미널 AI 어시스턴트)를 활용하여 설치를 진행합니다. Claude Code에게 명령어를 대신 실행시키거나, 각 단계에서 도움을 받을 수 있어 초보자도 쉽게 따라할 수 있습니다. 또한 OpenClaw onboard에 필요한 Anthropic 토큰 생성을 위해 Claude Code가 필수입니다.

관리자 권한 PowerShell 열기

중요: 이후 모든 명령어는 관리자 권한 PowerShell에서 실행합니다. 일반 PowerShell에서 실행하면 나중에 Gateway 서비스 설치가 실패하여 처음부터 다시 해야 합니다.
  1. 시작 메뉴 클릭 (또는 Windows 키)
  2. "PowerShell" 검색
  3. "Windows PowerShell"을 우클릭
  4. "관리자 권한으로 실행" 선택
  5. "이 앱이 디바이스를 변경할 수 있도록 허용하시겠어요?" -> 예 클릭
  6. PowerShell이 열리면 아래 명령어를 가장 먼저 실행하세요:
필수! PowerShell 실행 정책 설정 — 이걸 안 하면 이후 모든 명령어가 차단될 수 있습니다.
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

"실행 정책을 변경하시겠습니까?" 물어보면 Y 입력 후 Enter. 이 설정은 한 번만 하면 됩니다.

참고: Git 설치 후에는 PATH 갱신을 위해 PowerShell을 한 번 닫고 다시 열어야 합니다. 다시 열 때도 반드시 관리자 권한으로 여세요.
2
텔레그램 봇 생성 (먼저!)
왜 먼저? onboard 과정에서 봇 토큰을 입력해야 합니다. 미리 만들어두면 설정 중 중단 없이 진행할 수 있습니다.

단계별 안내

  1. 텔레그램 앱을 엽니다 (모바일 또는 데스크톱)
  2. 검색창에 @BotFather를 검색하고 대화를 시작합니다
  3. /newbot 명령어를 입력합니다
  4. 봇 이름을 입력합니다 (표시 이름, 예: My OpenClaw Bot)
  5. 봇 username을 입력합니다 (반드시 bot으로 끝나야 함, 예: my_openclaw_bot)
  6. BotFather가 봇 토큰을 발급해줍니다

토큰 형태 예시:

Bot Token 
1234567890:ABCdefGHIjklMNOpqrSTUvwxYZ123456789
중요: 이 토큰을 복사해서 메모장 등에 임시 저장해두세요! 토큰은 비밀번호와 같으므로 외부에 공유하지 마세요.
3
Git 설치

Claude Code 설치에 Git이 필요합니다.

이미 Git이 설치되어 있다면 이 단계를 건너뛰세요.

공식 사이트에서 설치

  1. https://git-scm.com/install/windows에 접속
  2. windows - Standalone Installer - "Git for Windows/x64 Setup." 클릭
  3. 다운로드된 설치 파일 실행
  4. 설치 마법사에서 "Next" 클릭하여 진행 (select components에서 Add a Git Bash Profile to Windows Terminal 체크)
  5. 환경변수 -> 시스템 변수 -> Path에 C:\Program Files\Git\cmd 포함 확인

PowerShell 재시작 및 설치 확인

중요: Git 설치 후 PowerShell을 닫고 새로 열어야 합니다! (다시 관리자 권한으로 여세요)
PowerShell (관리자) 
git --version

버전 번호(예: git version 2.53.0.windows.1)가 출력되면 성공입니다.

4
Claude Code 설치
이미 Claude Code가 설치되어 있다면 Anthropic setup-token 생성만 진행하세요.

Claude Code는 Anthropic에서 만든 터미널 기반 AI 어시스턴트입니다. 터미널에서 직접 AI에게 명령어 실행, 파일 관리, 설치 등을 맡길 수 있습니다.

참고: Claude Code는 OpenClaw 설치에 필수입니다. OpenClaw onboard 과정에서 필요한 Anthropic 토큰을 claude setup-token 명령어로 생성해야 하기 때문입니다.

사전 요구사항

  • Git: 3단계에서 설치 완료되어 있어야 합니다
  • Anthropic 계정: console.anthropic.com에서 가입

설치 방법

PowerShell (관리자) 
irm https://claude.ai/install.ps1 | iex

설치 완료 후 C:\Users\사용자이름\.local\bin\claude.exe 경로에 설치됩니다.

중요한 단계: 설치 후 PATH에 추가해야 합니다. 아래 명령어를 실행하세요:
PowerShell (관리자) 
[Environment]::SetEnvironmentVariable("PATH", "$([Environment]::GetEnvironmentVariable('PATH', 'User'));$env:USERPROFILE\.local\bin", "User")
설정 확인: 환경변수 -> 사용자 변수 -> Path에 C:\Users\사용자이름\.local\bin 포함되어 있으면 정상 설치 완료.

초기 설정

설치 후 PowerShell을 닫고 새로 열어야 합니다 (관리자 권한으로). 그 후:

PowerShell (관리자) 
claude

처음 실행하면 브라우저가 열리고 Anthropic 계정 인증을 요청합니다. 안내에 따라 로그인하면 됩니다.

설치 확인

PowerShell (관리자) 
claude --version

버전 번호(예: 2.1.42)가 출력되면 설치 성공입니다.

Anthropic setup-token 생성

중요: 이 토큰은 OpenClaw onboard 과정(7단계)에서 반드시 필요합니다! 반드시 이 단계에서 미리 생성해두세요.
PowerShell (관리자) 
claude setup-token
  1. 실행하면 브라우저가 열립니다
  2. Anthropic 계정으로 로그인합니다
  3. 인증 완료 후 터미널에 sk-ant-oat01-... 형태의 긴 토큰이 표시됩니다
  4. 이 토큰을 반드시 복사해서 메모장에 저장하세요!
이 토큰을 저장하지 않으면 onboard 진행 중에 터미널을 전환해야 하는 번거로움이 생깁니다.

Claude Code 활용 팁

  • "Node.js 설치해줘" -> 자동으로 설치 및 확인
  • "openclaw 설치해줘" -> 명령어 실행 및 오류 자동 대응
  • "이 화면 어떻게 해?" -> 스크린샷을 보여주면 다음 단계 안내
5
Node.js 설치

OpenClaw은 Node.js 기반이므로 Node.js를 설치해야 합니다.

방법 1: Claude Code에게 맡기기 (권장)

Claude Code 터미널에서 다음과 같이 요청하세요:

Claude Code 
Node.js 설치해줘

방법 2: 직접 명령어 입력

PowerShell (관리자) 
winget install OpenJS.NodeJS.LTS
  • 약관 동의 메시지가 나오면 Y를 입력합니다
  • 설치 중 추가 관리자 권한 팝업이 뜨면 "예"를 클릭합니다
  • 설치 성공 메시지가 나올 때까지 기다립니다

방법 3: 공식 사이트에서 설치

  1. https://nodejs.org/ko/download에 접속
  2. Windows 설치 프로그램(.msi) 버전 다운로드
  3. 다운로드된 .msi 파일 실행
  4. 설치 마법사의 안내에 따라 "Next"를 클릭하여 설치 완료

설치 확인

중요: Node.js 설치 후에는 반드시 PowerShell을 닫고 새로 열어야 합니다! (다시 관리자 권한으로 여세요)
PowerShell (관리자) - 새로 열기 
node --version
npm --version

버전 번호(예: v24.13.1 / 11.8.0)가 출력되면 설치 성공입니다.

npm 버전 번호가 안 나오고 오류가 뜨면? PowerShell에서 Set-ExecutionPolicy RemoteSigned 실행(변경옵션은 A) 후 재시도하세요.

OpenClaw 설치

방법 1: Claude Code에게 맡기기 (권장)

Claude Code 
https://openclaw.ai/ 설치해줘

방법 2: 직접 명령어 입력

주의: 반드시 --ignore-scripts 옵션을 붙여서 실행하세요! 이 옵션 없이 실행하면 node-llama-cpp 빌드 오류가 발생하여 설치가 실패합니다. Cloud AI(Claude, GPT 등)를 사용할 것이라면 이 옵션을 붙여도 아무 문제 없습니다.
PowerShell (관리자) 
npm i -g openclaw --ignore-scripts

설치에 상황에 따라 시간이 많이 걸릴 수 있으나 인내심을 가지고 완료될 때까지 기다려 주셔야 합니다.

성공 시 출력 예: added 672 packages in 41s

설치 확인

PowerShell (관리자) 
openclaw --version

버전 번호(예: 2026.2.12)가 출력되면 설치 성공입니다.

--ignore-scripts 없이 설치하고 싶다면? 로컬 AI 모델(llama 등)을 사용하려면 Visual Studio Build Tools (C++ 빌드 도구 포함)와 CMake를 먼저 설치해야 합니다. 대부분의 사용자는 Cloud AI를 사용하므로 --ignore-scripts로 충분합니다.

봇 초기 대화 (TUI)

TUI(터미널 UI)가 열리면서 봇이 인사합니다. 자유롭게 한국어로 대답하세요:

너 이름은 아라곤이야. 나는 sangs라고 불러줘.
타임존은 한국(KST)이야.
역할은 친절한 한국어 비서로 해줘.
TUI를 종료하려면: Ctrl + C를 누릅니다.
8
텔레그램 페어링 (연결 인증)
이 단계를 빠뜨리면 텔레그램에서 봇이 응답하지 않습니다!

봇 설치는 끝났지만, 텔레그램 계정과 봇을 연결(페어링)해야 대화할 수 있습니다.

  1. 텔레그램 앱에서 만든 봇을 찾아 /start 메시지를 보냅니다
  2. 봇이 다음과 같은 메시지를 응답합니다:
텔레그램 봇 응답 
OpenClaw: access not configured.

Your Telegram user id: 1234567890
Pairing code: ABCD1234

Ask the bot owner to approve with:
openclaw pairing approve telegram ABCD1234
  1. PowerShell에서 봇이 알려준 명령어를 그대로 실행합니다:
PowerShell (관리자) 
openclaw pairing approve telegram ABCD1234
주의: ABCD1234 부분을 텔레그램에서 받은 실제 페어링 코드로 바꿔야 합니다!
  1. 성공 시: Approved telegram sender 1234567890.
  2. 텔레그램에서 봇에게 아무 메시지나 보내보세요. AI가 응답하면 성공!
설치 완료 확인

모든 설정이 끝났습니다! 아래를 확인하세요:

  • openclaw --version 으로 버전 확인 가능
  • 텔레그램에서 봇에게 메시지를 보내면 AI가 응답
  • Gateway가 백그라운드에서 실행 중 (PC를 재시작해도 자동 실행)
관리자 권한 PowerShell은 이제 닫아도 됩니다. Gateway 서비스가 Windows 예약 작업으로 등록되어 백그라운드에서 계속 실행됩니다.

문제 해결 (Troubleshooting)

Node.js 또는 OpenClaw 명령어를 찾을 수 없음

증상
node: command not found
# 또는
openclaw: 이 시스템에서 스크립트를 실행할 수 없습니다

원인: 설치 후 터미널의 PATH 환경변수가 갱신되지 않음

해결: 현재 PowerShell 창을 완전히 닫고, 새 PowerShell을 관리자 권한으로 다시 열기

npm i -g openclaw 실행 시 빌드 오류

증상
npm error [node-llama-cpp] Failed to build llama.cpp
npm error code ENOENT

원인: node-llama-cpp 네이티브 모듈 빌드에 C++ 빌드 도구가 필요

해결:

PowerShell (관리자) 
npm i -g openclaw --ignore-scripts

Gateway service install failed

증상
Gateway service install failed.
Run PowerShell as Administrator or rerun without installing the daemon.

원인: 관리자 권한 없이 실행

해결: PowerShell 닫기 -> 시작 메뉴 -> "PowerShell" 검색 -> 우클릭 -> 관리자 권한으로 실행 -> openclaw onboard 재실행

Health check failed: gateway closed

증상
Health check failed: gateway closed (1006 abnormal closure)
Gateway: not detected

원인: Gateway 서비스가 설치되지 않았거나 실행되지 않음

해결: 위의 Gateway service install failed 해결 방법과 동일

텔레그램에서 access not configured 메시지

원인: 텔레그램 계정과 봇이 아직 페어링되지 않음

PowerShell (관리자) 
openclaw pairing approve telegram [봇이 알려준 페어링코드]

텔레그램에서 봇이 응답하지 않음 (Gateway 중지)

원인 확인:

PowerShell 
openclaw status

출력에서 Gateway가 중지된 상태인지 확인:

증상
Gateway: unreachable (connect ECONNREFUSED 127.0.0.1:18789)
Gateway service: stopped (state Ready)

해결:

PowerShell 
openclaw gateway start

정상 복구 확인: openclaw health 실행 후 Telegram: ok가 표시되면 정상입니다.

텔레그램에서 "Telegram already configured" 메시지

증상
Telegram already configured. What do you want to do?
  > Modify settings
    Skip (leave as-is)

원인: 이전에 이미 텔레그램 설정을 완료함

해결: 설정을 변경할 필요가 없으면 Skip (leave as-is) 선택

# 1. 텔레그램 봇 생성 (사전 준비)
